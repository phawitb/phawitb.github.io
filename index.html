Edge-LLM Inference with Sharded Model

Developed a distributed large language model (LLM) inference system by splitting the DistilGPT2 model into five sequential parts using PyTorch. Four parts were hosted on a PC (simulating edge devices) and one part ran on a real Raspberry Pi 5. Implemented socket-based tensor transfer with size headers for reliable networking between devices, enabling step-by-step collaborative text generation across heterogeneous hardware.

This study proposes an Internet of Things (IoT) network specifically designed to monitor greenhouse gases (GHGs) in mangrove forests. The network employs LoRa technology to transmit environmental data from sensor nodes to the central hub. LLM Fine-Tuning on Edge Devices with LoRA

Implemented and compared full fine-tuning and Low-Rank Adaptation (LoRA) techniques for the DistilGPT2 model on a Raspberry Pi 5 (4 GB RAM) using a custom Wikipedia-derived Solar System corpus. Evaluated training time, parameter efficiency, and inference performance, showing LoRA reduced trainable parameters to 0.18% of the model and cut training time per epoch by more than half while maintaining competitive loss reduction. LLM Fine-Tuning Using Layer-Wise Sharding

Implemented a distributed fine-tuning framework for the DistilGPT2 model (~82M parameters) by splitting its six transformer layers into five sequential shards deployed across Raspberry Pi and Ubuntu-based devices. Designed a TCP-based protocol for tensor exchange and backpropagation across shards, enabling end-to-end gradient updates on all devices. The server handled embeddings, final layers, and loss computation, while each shard trained its assigned transformer blocks locally. Logged performance metrics, memory usage, and loss curves, demonstrating stable cross-device training with persistent weight saving.

Autonomous Mobile Robot and Intelligent Systems

I have built a custom 2-wheels robot from scratch (using an Kinect+Arduino+Rotary Eecoders+IMU), which can navigate a predefined map autonomously, and also follow a person depending on the color of shirt he/she is wearing. For state estimation (odometry) of the robot, we used rotary encoders found on the wheels. We then integrated an IMU to get more accurate state estimation. For path planning, we used the A* algorithm to find the shortest path between the initial position and the goal. For path tracking, we used a PID controller. We mounted a Kinect camera on the robot for get enviromment data. We used Robot Operating System (ROS) for communication between all the parts of the project. This project was part of my Bachelor's degree research at Chulachomklao Royal Military Academy
